Tools

Node

Node is a runtime enviroment basen on javascript, but can run on a server

There are a few command associated with node which are import

node -v which tells what version of node you're running

node this command runs the selected file once

Package.json

This JSON file is located in the root of your project. It contains the metadata of your project. The package.json file is also our buildtool, in which we can declare scripts to run. It is also used to declare the dependencies needed in that specific project. If you're running an express project you need to declare that there.

NPM (Node Package Manager)

npm is used to install different packages into your node project. Using the command npm i followed by the package you need, in our case express, and it will be installed right in your package.json dependencies.

In some cases it will be nicer to install something globally on your computer, rather than in the different projects. Thats the case with Nodemon, in which you instead use the command npm install -g nodemon (-g for global)

Nodemon

Nodemon is a package used as a developer tool, since it can reload your node project after every change made. This is a great tool, since you only have to type nodemon instead of running node everytime a change is commited.

If you want to customise what nodemon listens on, you can create a nodemon.json file at the root of your project. Here you specify the file extension in which you want changes listened to.

Express

Express is a node web application framework, which helps you build a backend for a fullstack application using node and express.

Before getting started with express make sure you've downloaded npm and node.

To install express you need an empty node project, in which you create a file called package.json. In here simply put { } and run the command npm i express

Express will then be installed and you can then create your first express project

This is an example of a simple web application using express

            const express = require("express"); //This imports express via the require keyword
                const app = express(); //This initializes express and saves it as an variable called app, which can then later be used
                
                //Create an endpoint with app.get
                app.get("/", (req, res) =>{
                    res.send("Hello World");    //Send a string to the web application
                })
                
                //Open the server on a port of your choosing
                app.listen(3000, () =>{ 
                    console.log("Server is running on port ", 3000);
                });
fs

fs stands for file system and is used to read from and write to files.

        const fs = require("fs"); // import fs via require
            const nav = fs.readFileSync("./public/components/nav.html").toString(); // use fs to read the nav.html file
Heroku

Heroku is a cloud platform used to deploy all kind of web applications. It is free to use and easy to deploy unto.

Before use, navigate to the heroku site, and download their CLI. This will allow you to deploy on their site, with a few tweaks to your code and a couple of terminal commands.

The first thing you need to do is add a script into your package.json file:

            "scripts": {
                "start": "node "
            }

The next step is to change the port your application listens to, so change app.listen in app.js to the following

                app.listen(process.env.PORT || 3000, function(){
                    console.log("Express server listening on port %d in %s mode", this.address().port, app.settings.env);
                });

This was the tweaks needed in your code, now make sure your project is pushed so youre up to date

Run the command heroku create to create the heroku app, and push your github repo to it git push heroku main

If the project is a subfolder on your github app run the following command instead git subtree push --prefix heroku master

If you need to take your project down from heroku again simply run the following:

git remote rm heroku heroku git:remote -a
